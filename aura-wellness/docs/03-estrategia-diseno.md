## ♿ Accesibilidad

### Contraste de Color
- **Texto normal**: 4.5:1 mínimo
- **Texto grande**: 3:1 mínimo
- **Componentes interactivos**: 3:1 mínimo

### Tamaños Mínimos
- **Elementos táctiles**: 44px × 44px
- **Texto legible**: 14px mínimo
- **Iconos**: 24px × 24px

### Navegación
- **Orden de tabulación**: Lógico y predecible
- **Indicadores de foco**: Visibles y distintivos
- **Skip links**: Para navegación rápida

## 📱 Componentes

### Botones
- **Filled Button**: Primary con background sólido
- **Outlined Button**: Secondary con borde
- **Text Button**: Tertiary sin background
- **Icon Button**: Acciones contextuales

### Cards
- **Elevated Card**: Con sombra para contenido principal
- **Outlined Card**: Con borde para contenido secundario
- **Filled Card**: Con background para destacar

### Campos de Texto
- **Filled Text Field**: Con background sutil
- **Outlined Text Field**: Con borde definido
- **Leading Icon**: Para contexto visual

### Diálogos
- **Alert Dialog**: Para confirmaciones críticas
- **Simple Dialog**: Para selecciones simples
- **Full-screen Dialog**: Para flujos complejos

## 🎯 Principios de Diseño

### 1. Jerarquía Clara
- Información organizada por importancia
- Elementos visuales que guían la atención
- Agrupación lógica de contenido

### 2. Consistencia
- Patrones repetibles en toda la aplicación
- Comportamiento predecible de componentes
- Terminología consistente

### 3. Simplicidad
- Eliminación de elementos innecesarios
- Enfoque en la tarea principal
- Reducción de carga cognitiva

### 4. Accesibilidad
- Diseño inclusivo por defecto
- Soporte para todas las capacidades
- Cumplimiento de estándares WCAG
